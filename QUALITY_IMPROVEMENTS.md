## 상세 설명

### Q-02 Self-Review 체크리스트
LLM이 기사 초안 생성 후 출력 전에 스스로 점검하는 체크리스트.  
추가 API 호출 없이 프롬프트만으로 품질 향상.

```
□ 헤드라인이 구체적·검증 가능한 주장을 하는가?
□ 첫 문단이 단독으로 트윗 가능한가?
□ body_en의 모든 숫자가 제공된 팩트에 있는가?
□ cultural_note가 서양 독자에게 실제로 필요한 정보인가?
□ "In recent years", "It is worth noting" 같은 클리셰가 없는가?
□ 연속 단락이 같은 단어로 시작하지 않는가?
```

### Q-03 content_type 자동 분류
LLM이 팩트를 보고 기사 유형을 스스로 결정.

| 유형 | 기준 |
|---|---|
| `breaking` | 신규 발표·차트 결과·발매일 확정 등 시간 민감 팩트 |
| `evergreen` | 문화 개념·브랜드 소개·입문 가이드 (시간 무관) |
| `analysis` | 트렌드 설명·비교·의미 분석 |
| `profile` | 아티스트·브랜드 심층 소개·마일스톤 |

웹사이트 카드에 유형 배지 표시.

### Q-04 reader_level 분류

| 레벨 | 대상 | 특징 |
|---|---|---|
| `general` | K-culture 입문자, 일반 독자 | 용어 풀어쓰기, 배경 설명 충실 |
| `fan` | 팬덤, K-culture 마니아 | 빠른 팩트 전달, 내부 용어 허용 |

### Q-05 global_reaction
팬덤 반응·해외 언론 픽업·소셜 반응 규모를 1-2문장으로 자동 포함.  
기사 본문 마지막 단락 앞에 별도 callout으로 표시.

### Q-06 이미지 키워드 순차 시도
현재: `keywords[:2]`를 합쳐 1회 검색  
개선: 키워드를 1개씩 순서대로 시도 → 첫 번째 결과 반환  
효과: 더 관련성 높은 이미지 선택 가능성 향상.

### Q-07 당일 동일 엔티티 중복 방지
같은 날 같은 카테고리에서 태그 2개 이상 겹치는 기사는  
기존 기사가 있으면 스킵.  
예: 오늘 BTS 기사 이미 있으면 두 번째 BTS 기사 발행 안 함.
